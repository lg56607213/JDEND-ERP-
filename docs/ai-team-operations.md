## 2. 부여한 권한 (요약) — 상세는 각 `.md`의 "권한과 책임" 섹션

- **ERP 개발자**: ERP 코드 전권. 오류 자율 수정(단 기록+총괄 보고). **변경마다 GitHub `origin/main`에 요약 커밋+push.** (push 권한 검증 완료)
- **홈페이지 개발자**: 마케팅 홈페이지 로컬 우선 구축→추후 도메인/서버 구매. 잠정 가격 책정(기획과 논의). CS 게시판(CS와 논의).
- **CS 담당자**: 고객만족 최우선. 정식 배포 후 매일 9시 불만 보고 + 주/월 분석(자동보고는 스케줄러 설정 필요).
- **마케팅 담당자**: 온/오프라인 마케팅 방안·로드맵. 타깃 렌터카사 리스트 취합(공개정보·적법성 유의).
- **기획자**: 경쟁사·서버비용·손익·이익극대화 분석, 분/반기/연 로드맵, 유입 분석.
- **QA 테스터**: **테스트 테넌트(test111/112/113)에 한해 CRUD 전권.** 실데이터 회사(erp_company_jdend 등)는 읽기만. 규모별 전수 테스트, 전표 누락·재무제표 정합성 점검, 실시간 이중 보고.

1. **결과물을 직접 확인하고 증거를 제시한다.**
2. **확인 못 한 건 "미확인/미검증"이라고 말한다.** (하지 않은 걸 했다고 하지 않는다)

## 3. ⏳ 사장 결정 대기 사안

### [#5] 차량 매각/취득 자산계상 회계모델 — **결정 필요**
현재 `VehicleSaleService.java`(매각 전표)에 문제:
- (a) 차량 취득 시 '차량운반구' 자산을 안 잡아 → 매각 시 자산이 음수가 됨
- (b) 대차를 억지로 맞추는 허수 계정 '미상각잔액' 사용
- (c) 감가상각누계액을 실제 전표분이 아닌 스케줄 기준으로 계산
- (d) 현금주의인데 미수(현금 미수령) 상태로 판매수익을 즉시 인식
- ※ (a)(b)는 "취득 시 자산 미계상"이라는 한 원인의 증상 → 취득 모델을 정해야 매각도 정리됨.

**선택지 (하나 고르면 erp-dev가 구현):**
- **A.** 차량 취득/등록 시 '차량운반구' 자산 계상 — 가장 정합적, 취득 흐름까지 변경
- **B.** 취득은 그대로, 매각 시 장부가 기준 처분손익만 인식 — 변경 작음, 신뢰할 장부가 출처 필요
- **C.** 현금주의 매각 — 현금 받을 때 인식(미수 아님), '매각 수납' 이벤트 필요
- (A+C 병행 가능)

### (참고) 아직 안 고친 기존 이슈 — 이번 수정과 별개, 추후 결정
- 자동전표가 '대기' 상태로 생성 → 승인 안 하면 재무제표·자금일보에서 누락 (승인 워크플로우가 의도된 설계인지 확인 필요)
- 신규 회사 생성 시 재무제표 기본 계정구조가 0건으로 복제되는 버그 (복제 원본 DB가 비어 있음 — 회사 PC 실데이터로 재확인 필요)
- 보안: 로그인 없이 API 접근 가능 (사장 지시로 현재 보류)
