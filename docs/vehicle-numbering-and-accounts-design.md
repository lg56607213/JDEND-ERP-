# 차량 취득 회계·번호체계 설계

> 결정일: 2026-07-11 (총괄 사장 결정). 이 문서가 구현 기준(계약)이다.

## 확정된 결정

1. **기타계정관리 = 재무제표관리 연동 (하드코딩 제거).**
   - 기타계정관리(총괄관리>회계계정관리)의 계정 드롭다운은 **재무제표관리에 등록된 계정만** 뜨고 선택 가능.
   - 재무제표관리(`/api/financial-statement-accounts`, 특히 `/leaves` 말단계정)가 **계정 마스터(단일 진실)**. 계정코드·계정명은 여기서만 온다.
   - 저장 시 선택한 계정코드가 재무제표관리에 존재하는지 **백엔드에서도 검증**(없으면 거부).

2. **선급렌트자산 미사용 (현 정책).**
   - 주 고객이 소규모(차량 먼저 구매)라, **선급 단계에서 '차량운반구'로 직접 등록**.
   - 선급렌트자산 방식(계약 먼저 → 차량 나중, 대형 장기렌트업체용)은 **추후 옵션**으로 남김.

3. **번호체계 (신규 생성분부터 적용, 기존 데이터 마이그레이션 안 함).**
   - 발주번호 = `J + YYMMDD + 순번3` → 예 `J260701001`
   - 차량관리번호 = `발주번호 + 순번3` → 예 `J260701001001`
   - **1발주 = 여러 대 가능**: 끝 3자리는 그 발주 안의 차량 순번(001, 002, …)
   - 차량관리번호는 **실행 시점에 채번(확정)**.

4. **차량관리번호 = 모든 차량 관련 전표의 공용 키(주민번호 격).**
   - 실행 후 감가상각·정비·보험·매각은 물론 **수납 전표도 차량관리번호로 기재**.

5. **실행 이관.**
   - 발주(선급) 때 생긴 전표를 실행 시 **차량관리번호로 이관**.
   - 선급을 차량운반구로 직접 잡으므로 **계정 대체는 불필요**, 전표 키를 **발주번호 → 차량관리번호로 갱신**.
   - (선급렌트자산을 쓰는 경우엔: 선급렌트자산→차량운반구 대체 + 세금과공과 등 부대비용도 차량관리번호로 이관 → 차대변 정합. 지금은 미사용.)

## 구현 단계

### Phase 1 — 지금 구현 (번호체계와 독립)
- 기타계정관리 프론트: 드롭다운을 재무제표관리 계정목록 API에서 **동적 로드**. 저장 시 계정코드 존재 검증(프론트+백엔드).
- 선급(`VehicleAdvanceService`): 기본 차변 = 차량운반구. 기타계정관리 매핑이 설정돼 있으면 **그 설정대로**(현재 항목명이 안 맞아 매번 기본값으로 새는 문제 수정).

> **구현 완료 (2026-07-11)**
> - `account_management.html`: 하드코딩 `<option>` 전부 제거 → 페이지 로드 시 `GET /api/financial-statement-accounts/leaves`로 계정목록 동적 로드. 선급 항목(prepaid-item)도 JS(`PREPAID_ITEMS`)로 렌더. 은행계좌 select은 `.bank-select`로 분리해 계정목록 주입에서 제외.
> - 저장 검증: `OtherAccountSettingsService.save` → `validateAccountCodesExist`가 settings 내 모든 `account` 코드를 `FinancialStatementAccountRepository.existsByAccountCode`로 검증, 미등록 코드는 저장 거부.
> - **항목명 불일치 원인 확인·수정**: 기타계정관리는 `인지세`, 선급화면(`vehicle_advance.html`)은 `인지대`로 달랐고 `외주장착비용` 항목이 기타계정관리에 없었음 → `PREPAID_ITEMS`를 선급화면 항목명과 정확히 일치시켜 매칭 복구. 미지정 항목은 `VehicleAdvanceService`가 기본값 `차량운반구`로 처리(현 정책, 선급렌트자산 미사용).
> - 참고: 기존 하드코딩 코드(2010 선급렌트자산·1110 차량운반구 등)는 실제 계정 마스터와 불일치했음. 마스터 실제 코드는 `100401 차량운반구`, `500211 세금과공과` 등이며 `선급렌트자산` 계정은 마스터에 없음.

### Phase 2 — 설계 잠근 뒤 구현
- 발주번호/차량관리번호 채번 로직을 새 체계로.
- 전표 공용키(차량관리번호) 확립 + 모든 차량/수납 전표에 기재.
- 실행 시 발주번호 → 차량관리번호 전표 이관.
- **확인 필요**: 기존 `vehicleNo`(차량번호=번호판?) vs `vehicleMgmtNo`(차량관리번호) 필드 관계 — 전표 공용키를 `vehicleMgmtNo`로 통일할지.
