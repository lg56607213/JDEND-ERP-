## 0. 이 문서를 쓴 근거 (직접 확인한 것 / 미확인한 것)

**직접 확인한 제품 기능** — `src/main/resources/static/pages/` 화면 기준:
- **차량**: 발주→출고→등록→실행 흐름(차량현황), 정비, 보험, 배차, 정기검사, 차량 매각, 감가상각.
- **계약**: 계약등록, 계약 총괄관리(진행중/만기/종료/반납 상태 배지), 만기관리, 만기해지·중도해지, 법적절차(법무).
- **회계**: 일일전표(자동전표 연동), 매출전표, 재무상태표·손익계산서(재무제표), 감가상각대장, 자금일보, 미지급금 관리.
- **청구/수납**: 청구서 생성·발행, 미수현황(미납/일부결제/완료 상태), 세금계산서 발행, 대출관리, 은행거래 업로드.
- **기타**: 고객관리, 세무상담 게시판.

**미확인(홈페이지에 사실처럼 쓰면 안 되는 것)**:
- 가격/요금제 — 아직 미확정(planning과 논의 필요). 아래 카피는 "상담 시 안내" 톤으로 처리.
- 도입 고객 수·절감 시간·정확도 같은 **수치 실적** — 초기 스타트업이라 검증된 데이터 없음. 수치 주장 전면 배제.
- 모바일 앱 지원 여부, 차량관제(GPS/시동차단) 여부 — 화면상 미확인이라 언급하지 않음.
- 데이터 이관/보안 인증(ISMS 등) 구체 사양 — 미확인. FAQ에서 "상담 시 안내" 톤으로만 처리.

> **표시광고 리스크 주의**: "국내 1위", "100% 자동", "최저가", "누구나 1분 완성" 같은 최상급·단정 표현은 근거가 없으므로 쓰지 않음. 아래 카피는 기능이 실제로 하는 일만 서술.

**경쟁 지형(WebSearch 확인)**: 렌트데스크·카모아 파트너스·올댓렌트카·롯데/SK렌터카 커넥트 등 다수는 **예약·차량관제·전자계약** 중심.
→ JDEND의 차별점은 **"계약부터 회계·재무제표까지 이어지는 정산/장부 자동화"**. 특히 **장기렌터카(여신·미수·만기 관리)** 사업자에게 맞춤.

## 1. 히어로 카피 (헤드라인 + 서브카피)

### A안 — "장부까지 끝나는" (회계 차별점 강조 · 추천)
- **헤드라인**: 계약을 넣으면, 장부까지 끝나 있습니다
- **서브카피**: 렌터카 계약·수납을 입력하면 전표와 재무제표가 자동으로 이어집니다. 차량 관리부터 미수·세금계산서까지 한 곳에서.
- **소구점**: 경리 담당이 이중 입력·수기 전표에 쏟던 시간을 없앤다.

### B안 — "흩어진 것을 한 곳으로" (통합 강조)
- **헤드라인**: 엑셀 열 개로 하던 렌터카 관리, 이제 한 화면에서
- **서브카피**: 차량·계약·수납·회계를 따로 관리하지 마세요. 발주부터 만기·정산까지 하나로 연결된 렌터카 전용 ERP.
- **소구점**: 엑셀·수기 장부의 누락과 대사(對査) 스트레스를 줄인다.

### C안 — "돈 흐름이 보인다" (사장의 손익 시야 강조)
- **헤드라인**: 이번 달, 우리 렌터카는 얼마 남았을까
- **서브카피**: 계약별 미수, 차량별 비용, 회사 손익까지. 입력한 데이터가 그대로 재무제표가 되는 렌터카 ERP.
- **소구점**: 사장이 매출·미수·손익을 한눈에 보고 판단한다.

> 권장: 메인은 **A안**(경쟁사 대비 유일한 차별점=회계 자동화), 스크롤 하위 배너/광고 소재에 **B·C안** 분산.

## 2. 기능별 소구 카피 (섹션 헤드라인 + 짧은 설명)

각 설명은 확인된 화면 기능만 서술. 경리·사장이 쓰는 말로.

### 2-1. 차량관리
- **헤드라인**: 발주부터 매각까지, 차 한 대의 일생을 한 줄로
- **설명**: 발주·출고·등록·실행 단계별 차량현황을 한 화면에서 봅니다. 정비·보험·정기검사·배차 이력이 차량별로 쌓이고, 매각과 감가상각까지 연결됩니다.

### 2-2. 계약관리
- **헤드라인**: 진행중·만기·해지, 계약 상태가 색으로 보입니다
- **설명**: 계약을 등록하면 총괄관리 화면에서 진행중/만기/종료/반납 상태를 한눈에 확인합니다. 만기·중도해지, 필요 시 법적절차까지 계약 흐름 그대로 처리합니다.

### 2-3. 자동 회계·전표
- **헤드라인**: 수납을 입력하면 전표가 알아서 만들어집니다
- **설명**: 계약·수납·지출을 입력하면 일일전표·매출전표가 자동으로 생성됩니다. 회계를 몰라도 렌터카 업무만 입력하면 장부가 남습니다.
  - *(주: 자동전표는 확인/마감 단계를 거치는 구조. 홈페이지에는 "자동 생성 후 확인" 톤으로 표기. 100% 무인 자동으로 과장하지 말 것.)*

### 2-4. 재무제표
- **헤드라인**: 마감 없이도, 오늘 기준 재무상태표·손익계산서
- **설명**: 입력한 전표가 그대로 재무상태표와 손익계산서로 이어집니다. 세무사에게 넘길 자료도, 사장이 볼 손익도 같은 데이터에서 나옵니다.
  - *(주: 회계기준은 현금주의. 필요 시 "현금 실수령 기준" 명시 가능.)*

### 2-5. 청구·수납 / 미수관리
- **헤드라인**: 누가, 얼마를, 언제까지 안 냈는지 바로 보입니다
- **설명**: 청구서 생성·발행부터 세금계산서까지 한 번에. 미수현황에서 미납·일부결제·완료 상태를 걸러 보고, 밀린 대금을 놓치지 않습니다.

### 2-6. 자금·정산 (보조 섹션)
- **헤드라인**: 오늘 들어온 돈, 나간 돈을 자금일보로
- **설명**: 일자별 자금 흐름과 미지급금을 정리해 회사의 현금 사정을 매일 확인합니다. (은행거래 내역 업로드 지원)

**폼 제출 버튼**
- 상담 신청하기 / 자료 받기 / 데모 요청하기

## 4. FAQ 후보 (10개)

> 답변은 **확정 정보가 없는 항목은 "상담 시 안내" 톤**으로 작성. 사장이 실제로 묻는 순서(가격→이관→난이도→보안→지원)로 배치.

**Q1. 가격이 어떻게 되나요? 차량 대수에 따라 다른가요?**
A. 요금은 회사 규모(차량 대수·사용 인원)에 맞춰 안내드립니다. 상담 시 현재 운영 규모를 알려주시면 맞는 방안을 제안해 드립니다.
*(주: 가격 미확정. planning 확정 후 실제 금액/구간으로 교체.)*

**Q2. 지금 쓰는 엑셀·기존 프로그램 데이터를 옮길 수 있나요?**
A. 보유하신 차량·계약·거래처 자료를 기준으로 이관 방법을 함께 정리해 드립니다. 자료 양식과 범위는 상담 때 확인합니다.
*(주: 이관 자동화 수준 미확인 — 과장 금지.)*

**Q3. 회계를 잘 몰라도 쓸 수 있나요?**
A. 렌터카 업무(계약·수납·지출)를 입력하면 전표가 자동으로 만들어지도록 설계했습니다. 회계 지식보다 평소 업무 흐름대로 입력하시면 됩니다.

**Q4. 세무사에게 넘기는 자료로 쓸 수 있나요?**
A. 입력 데이터로 재무상태표·손익계산서가 생성됩니다. 세무 대리인과 공유할 자료로 활용하실 수 있으며, 필요한 항목은 상담 때 확인해 드립니다.

**Q5. 데이터는 안전하게 보관되나요?**
A. 고객 회사별로 데이터를 분리해 관리합니다. 백업·접근 관리 등 구체 사항은 상담 시 안내드립니다.
*(주: 보안 인증/암호화 사양 미확인 — 인증 보유를 사실처럼 표기하지 말 것. 참고로 로그인 인증은 현재 보류 상태(내부 이슈)라 대외 보안 카피는 보수적으로.)*

**Q6. 단기렌터카·장기렌터카 둘 다 되나요?**
A. 계약·미수·만기 관리가 중심이라 **장기렌터카(월정액·여신형)** 운영에 특히 잘 맞습니다. 운영 형태를 알려주시면 적합 여부를 함께 확인해 드립니다.
*(주: 단기 예약/관제 기능은 화면상 미확인 — 단기 예약 특화로 홍보하지 말 것.)*

**Q7. 컴퓨터를 잘 못 다뤄도 되나요? 설치가 복잡한가요?**
A. 웹 브라우저에서 사용하도록 만들었습니다. 도입 시 화면 안내와 초기 설정을 함께 도와드립니다.
*(주: "설치 없음" 여부는 배포 방식 확정 후 반영.)*

**Q8. 직원 여러 명이 같이 쓸 수 있나요? 권한을 나눌 수 있나요?**
A. 회사 내 사용자 계정을 만들어 함께 사용할 수 있습니다. 사장·경리 등 역할에 따른 권한 구분은 상담 때 확인해 드립니다.
*(주: 사용자/권한 관리 화면 존재 확인. 세부 권한 정책은 개발 확인 후 확정.)*

**Q9. 도입하면 세팅과 교육은 어떻게 하나요?**
A. 초기 회사 정보·차량·계약 등록을 함께 진행하고, 실제 업무에 쓰실 수 있도록 사용법을 안내해 드립니다.

**Q10. 계약 기간이나 해지 조건이 있나요?**
A. 부담 없이 시작하실 수 있도록 조건을 안내드립니다. 자세한 사항은 상담 시 확인해 주세요.
*(주: 약정/해지 정책 미확정. 확정 후 정확히 기재 — 자동연장·위약금 등은 전자상거래법 표기 의무 있음.)*

## 6. 신뢰·전환 요소 아이디어 (초기 스타트업 기준)

검증된 실적·고객 로고가 없는 초기 단계이므로, **"실물을 보여주는 신뢰"**로 접근. 없는 것을 있는 척하지 않는다.

**A. 지금 당장 쓸 수 있는(사실 기반) 신뢰 요소**
1. **실제 화면 스크린샷/짧은 데모 영상** — 차량현황·계약 총괄·재무제표·미수현황 실제 화면. "말"보다 "화면"이 초기 신뢰의 핵심.
2. **기능 체크리스트/비교표** — "엑셀 vs JDEND"로 무엇이 자동화되는지 표로. (경쟁사 실명 비교는 표시광고 리스크 → 회피)
3. **개발사가 직접 응대** — "만든 사람이 직접 상담·세팅합니다" 같은 1인/소수 정예의 강점을 밀착 지원으로 전환.
4. **회계 정합성 소구** — 현금주의 기준으로 전표·재무제표가 맞도록 설계·검증 중임을 담담하게(과장 없이).

**B. 리드 수집 퍼널(전환) 설계 제안**
1. 히어로 CTA(상담) → 2. 기능 스크롤 → 3. FAQ로 이의 해소 → 4. 하단 반복 CTA + 간단 폼(회사명/대수/연락처/현재 사용 도구).
   - 폼은 **필드 최소화**(4개 이내). 대수 정보는 후속 견적/응대에 유용.
2. **오프라인 연계**: "회사소개서·도입제안서 우편/이메일 받기" 옵션 → 우편 발송 리스트와 연결(마케팅팀 오프라인 과제와 통합).
3. **당근·네이버 유입** → 홈페이지 랜딩 → 상담 폼. 채널별 UTM으로 유입 출처 추적(planning 유입 분석과 연계).

**C. 초기에 만들면 좋은(향후) 신뢰 자산**
- 첫 도입 고객의 **후기/사례**(확보되는 대로. 없는 후기 창작 금지).
- **자주 쓰는 렌터카 업무 가이드/블로그** — 전문성 신뢰 + SEO 이중 효과.
- 사업자정보·대표자·연락처·환불/이용약관 명시(전자상거래법 표시의무 + 신뢰).

## 7. homepage-dev / planning 에게 넘기는 확인 요청 (건의)

카피를 사실로 확정하려면 아래가 필요:
1. **가격/요금제·약정·해지 정책** (planning) → FAQ Q1·Q10, 가격 페이지 확정.
2. **배포 방식**(웹 SaaS/설치형) → "설치 없이 웹에서" 문구 사용 가부.
3. **데이터 이관 지원 수준** → FAQ Q2 문안 강도.
4. **보안 현황**(현재 로그인 인증 보류 이슈 있음) → 보안 카피는 확정 전까지 보수적으로. 정식 오픈 전 인증/접근제어 정리 권장(건의).
5. **모바일 지원·단기 예약/관제 기능 유무** → 있으면 소구 섹션 추가, 없으면 언급 제외.
6. **표시광고 검토**: 정식 오픈 전 "무료/최저/1위" 류 표현 최종 점검(전자상거래·표시광고법).
