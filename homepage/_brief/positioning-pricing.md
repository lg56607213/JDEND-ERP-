## 0. 먼저 — 제품이 무엇인가 (포지셔닝의 출발점)

코드(`src/main/resources/static/pages/`)를 확인한 결과, 이 제품은 단순 "렌터카 예약/배차 관리 프로그램"이 아니다. 다음을 한 시스템에서 다룬다:

- **차량 생애주기**: 발주 → 선급 → 출고/실행 → 배차 → 정비 → 보험 → 정기검사 → 매각
- **계약**: 계약등록/조회/출력, 만기관리, 중도해지/만기해지, 법적절차(연체 소송)
- **자금·여신**: 차입금(loan) 등록/스케줄/조정, 연체관리, 여신
- **청구·수납**: 청구서 발행, 수납, 미수금 관리, 세금계산서 발행, 은행거래 업로드
- **회계**: 전표(일/월/매출), 전표승인, 감가상각(등록/스케줄변경), 재무제표(재무상태표·손익계산서), 자금일보
- **멀티회사(멀티테넌트)**: 회사별 분리 운영

즉 경쟁 카테고리가 두 개다:
1. **렌터카 예약/배차 관리 SW**(렌트데스크·투어오피스·올댓렌트카 등) — 우리는 이걸 포함하고 그 위에 **회계·재무제표·감가상각·차입금/여신**까지 얹음.
2. **범용 ERP/회계**(이카운트 등) — 우리는 이걸 **렌터카 도메인에 특화**(차량관리번호 공용키, 감가상각·매각·여신 자동전표)했다는 게 차별점.

**전략적 함의(솔직하게):** 우리의 진짜 강점은 "차량을 **자산으로 소유·금융(차입)** 하고, 그 감가상각·처분손익·재무제표까지 굴려야 하는 회사"에게 크다. 단순히 예약만 받는 초소형 매장에는 이 강점이 과할 수 있다. → **우선 공략은 소규모 최말단이 아니라 '중형(자가 보유 + 차입 운용)'이 합리적.** (근거: 아래 2·3장)

## 1. 타깃 세그먼트

세그먼트 규모 구분은 사장이 지정한 테스트 페르소나 기준(소규모<50 / 중형 200~300 / 대형 300~500)을 따른다.

### A. 소규모 (< 50대)
- **현실**: 사장+직원 1~3명. 관리 대부분 **엑셀·수기·카톡**, 세무는 외부 기장(월 5~15만원 가정, 미확인 범위). 예약관리 SW만 쓰거나(렌트데스크 등 월 5만원) 아무것도 안 씀.
- **페인포인트**: (1) 계약·수납·연체가 엑셀로 흩어져 미수금 누락. (2) 차량별 손익(감가상각·정비비)을 모름. (3) 세금계산서·전표를 손으로. 
- **지불의사(WTP)**: 낮음·가격 민감. 월 5~10만원이 심리 저항선(가정, 경쟁 예약SW 5만원 기준).
- **우선순위**: **2순위.** 수가 가장 많아 볼륨은 되지만, 우리 차별점(여신·재무제표)의 가치를 덜 느끼고 이탈 위험·CS 부담이 큼. → **라이트 요금제로 유입 → 성장하면 상위 티어로 업셀** 하는 깔때기 입구로 활용.

### B. 중형 (200~300대)  ← **우선 공략 1순위**
- **현실**: 차량을 **캐피탈/은행 차입으로 대량 취득**해 운용. 직원 5~15명. 이미 예약SW + 별도 회계(이카운트/세무사) + 엑셀을 **병행**해 데이터가 3~4곳에 흩어짐.
- **페인포인트**: (1) **차입금 상환 스케줄·이자**와 차량 손익이 따로 놀아 실제 대당 수익을 모름. (2) 감가상각·처분손익 수기 계산 → 재무제표 신뢰 낮음. (3) 계약 만기·재렌트·중도해지 관리가 사람에 의존. (4) 여러 시스템 간 이중입력.
- **지불의사**: 중간~높음. 여러 SW를 하나로 합쳐 **월 20~50만원**을 아끼거나 대체할 수 있으면 지불 여력 있음(가정).
- **우선순위**: **1순위.** 우리 차별점(차량관리번호로 계약~수납~감가~매각~여신을 하나의 키로 묶는 자동전표, 재무제표 연동)이 **정확히 이들의 고통을 푼다.** CS 밀도 대비 매출 단가가 좋음.

### C. 대형 (300~500대+)
- **현실**: 전산 담당·회계팀 존재. 자체 시스템이나 SI 구축 SW를 쓰는 경우 많음. 멀티 법인/지점.
- **페인포인트**: (1) 낡은 SI 시스템 유지보수비·경직성. (2) 멀티회사 통합 재무 가시성. (3) 규제·감사 대응.
- **지불의사**: 높지만 **의사결정 느리고 요구 많음**(커스터마이즈·보안·SLA). 1인 창업 초기엔 대응 리소스 부족.
- **우선순위**: **3순위(초기), 후기 성장 타깃.** 멀티회사 기능이 이미 있어 레퍼런스로서 가치 큼 → **첫 1~2곳은 전략 고객(할인+밀착지원)** 으로 확보해 신뢰 자산화. 단, 초기에 여기 올인하면 커스터마이즈 늪에 빠짐 → **선별 수주**.

> **요약 우선순위: 중형(1) → 소규모(2, 깔때기) → 대형(3, 선별 레퍼런스).**

## 2. 핵심 가치제안 (왜 이 ERP인가)

### 한 줄 메시지 (홈페이지 히어로용 제안)
> **"차량 한 대의 발주부터 매각까지, 계약·수납·차입금·회계·재무제표를 하나로."**
> — 렌터카에 딱 맞춘 ERP. 엑셀 3개, 회계 프로그램 따로 쓰던 걸 한 시스템에서.

### 차별점 (기존 방식·경쟁 대비)

| 비교 대상 | 그들의 방식 | 우리 차별점 |
|---|---|---|
| **엑셀·수기** | 계약·수납·차량·차입금이 파일 3~4개로 흩어짐. 재입력·누락·계산오류. | **차량관리번호 하나로** 계약~수납~정비~감가상각~매각~여신이 전부 연결. 자동전표. |
| **예약/배차 SW** (렌트데스크·투어오피스 등) | 예약·배차·정비까지는 되나 **회계·재무제표·감가상각·차입금은 없음** → 세무사·엑셀 병행. | 예약·배차 + **회계·감가상각·재무제표·차입금까지 한 시스템.** 데이터 이중입력 제거. |
| **범용 ERP** (이카운트 등) | 회계·재고는 강하나 **렌터카 도메인 지식 없음**(차량 생애주기·재렌트·중도해지·감가·매각 처분손익을 수작업 설계해야 함). | 렌터카 **업무 흐름과 전표 규칙이 기본 내장**. 도입 즉시 렌터카 회계가 돌아감. |
| **SI 구축 시스템** (대형사) | 초기 수천만원, 유지보수 경직, 변경 느림. | 구독형(월정액), 지속 업데이트, 초기비용 낮음. |

### 구체 효용(카피 소재)
- "차량별 실수익을 본다" — 취득원가·감가상각·정비비·차입이자 vs 렌트수익을 대당으로.
- "미수금이 안 샌다" — 청구·수납·연체·법적절차가 한 흐름.
- "세금계산서·전표가 자동" — 수납하면 전표가 생김(현금주의 기준).
- "만기·재렌트를 놓치지 않는다" — 계약 만기·중도해지·재렌트 관리.

> **주의(과장 금지):** "완전 자동 회계"라고 쓰지 말 것. 현재 자동전표가 '대기' 상태로 생성돼 승인 워크플로우가 필요할 수 있고(운영 문서 #참고 이슈), 차량 취득/매각 회계모델(#5)은 결정 대기 중. 홈페이지엔 **"수납·감가상각·매각 시 전표를 자동 생성"** 정도의 검증된 표현만 사용.

## 3. 요금제 전략 (가격 근거 포함)

### 3-1. 시장 가격 레퍼런스 (조사)
| 제품 | 카테고리 | 가격 | 비고 |
|---|---|---|---|
| 렌트데스크 | 렌터카 예약/차량관리 | **월 5만원** | 회계·재무제표 없음 |
| 투어오피스 | 렌터카 예약관리 | **월 9만원~**(VAT별도) | 웹호스팅+유지보수 |
| 이카운트 ERP | 범용 ERP/회계 | **월 4만원** + 가입비 20만원 (사용자·거래 무제한) | 렌터카 특화 아님 |
| 통합 ERP(일반) | 대기업형 | **월 30만~200만원** | SI/구축형 |

(출처는 문서 하단.) → 우리 제품은 **예약SW(5~9만원)와 범용ERP(4만원)를 합친 것 이상**의 렌터카 특화 통합. 따라서 **예약SW보다 높고, 통합ERP 하단보다 낮은** 구간이 타당.

### 3-2. 원가 구조 (손익 근거) — 가정 명시
- **한계원가는 서버비 중심**(1인 창업 + AI팀이라 인건비 고정, 고객 1곳 추가의 변동비는 사실상 서버·DB).
- 멀티테넌트: **회사당 DB 분리**(템플릿 DB 복제). → 테넌트당 스토리지·커넥션 비용 발생하나, 클라우드 소형 인스턴스에 다수 수용 가능.
- **가정(미확인, 실측 필요):** 클라우드 VM+관리형 DB 월 10만~30만원대에서 초기 수십 테넌트 수용 가능 → **테넌트당 실서버 원가 대략 월 수천~1만원 수준**으로 추정. 이 경우 **구독가 대비 그로스 마진 90%+** (SaaS 전형).
- 결론: **가격은 원가가 아니라 "가치·경쟁·지불의사"로 정한다.** 원가는 하한(대당 최소가)만 방어하면 됨. → **초기 서버 실비를 측정해 확정할 것(액션: 총괄/개발과 실측).**

### 3-3. 제안 요금제 (차량 대수 기반 월 구독) — 잠정

기본 원리: **"기본 월정액 + 차량 대수 포함분 + 초과분 대당 과금."** 대수 늘수록 대당 단가↓(볼륨 할인). 연납 시 2개월분 할인(약 16%) 제안.

| 티어 | 대상 | 포함 차량 | **월정액(잠정)** | 초과 대당 | 주요 범위 |
|---|---|---|---|---|---|
| **Lite** | 초소형 <20대 | 20대 | **월 6만원** | +3,000원/대 | 차량·계약·수납·정비 (회계 기본) |
| **Basic** | 소규모 ~50대 | 50대 | **월 12만원** | +2,500원/대 | + 감가상각·재무제표·세금계산서 |
| **Standard** | 중형 50~300대 | 200대 | **월 39만원** | +1,500원/대 | + 차입금/여신·연체·법적절차·자금일보 |
| **Pro** | 중대형 300~500대 | 500대 | **월 79만원** | +900원/대 | + 멀티회사·우선지원 |
| **Enterprise** | 500대+ / 멀티법인 | 협의 | **별도 견적** | 협의 | 전용 지원·SLA·데이터이관 밀착 |

**대당 단가 정합성 체크(왜 이 숫자인가):**
- 50대 Basic = 12만원 → 대당 2,400원. 예약SW(5만원)+회계 대체 가치 감안 시 수용 가능선(가정).
- 200대 Standard = 39만원 → 대당 1,950원. 여러 SW/세무 병행 비용(월 30~50만원 가정) 대체 → **비용 절감 소구** 가능.
- 500대 Pro = 79만원 → 대당 1,580원. 통합ERP 하단(30만~) 위지만 SI 구축형(수천만원)보다 훨씬 쌈.

**가격 결정의 솔직한 트레이드오프:**
- **Lite/Basic이 예약SW(5만원)보다 비싸 보이는 문제** — 반드시 카피로 "예약SW + 회계프로그램 + 세무 일부를 하나로"라고 **묶음 가치**를 보여줘야 함. 안 그러면 소규모는 "렌트데스크가 더 싼데?"로 이탈.
- **소규모는 가격 민감** → Lite는 유입/체험용 미끼(로스리더 성격). 마진은 Standard 이상에서 확보.
- 초과 대당 과금은 **성장하는 고객이 자연스럽게 매출을 늘려주는 구조**(net revenue retention). 좋은 설계.

### 3-4. homepage-dev 잠정가 대체 지침
- 홈페이지 요금제 페이지에는 위 **5티어 + 월정액**을 노출.
- 초과 대당 단가·연납 할인은 "문의 시 안내" 또는 작은 글씨로.
- **"모든 요금 VAT 별도" 명시**, 가입비는 **없음**(이카운트와 차별화 포인트로 활용 가능 — "가입비 0원").
- Enterprise는 가격 대신 **"문의"** CTA.
- **미확정 표기:** 실서버 원가 실측 전이라 가격은 **베타 프로모션가로 시작 가능**("정식 출시 전 얼리버드 30% 할인" 등) — 초기 레퍼런스 확보용으로 권장.

## 4. 도입/온보딩 절차 (제안)

홈페이지 "도입 절차" 섹션 + CTA 설계.

1. **① 문의/상담 신청** (폼: 회사명·차량대수·현재 사용 도구·연락처) → 대수로 티어 자동 추천.
2. **② 온라인 데모** (30분, 화면 공유) — 우리 화면으로 "당신 회사의 차량 1대가 발주→수납→재무제표까지 흐르는" 시나리오 시연.
3. **③ 무료 체험(가정: 14~30일)** — 테스트 테넌트 제공. 실제 차량 몇 대 넣어보게.
4. **④ 데이터 이관 지원** — 기존 엑셀/타 SW의 차량·계약·미수금을 초기 적재(초기엔 수동/반자동, 밀착 지원).
5. **⑤ 정식 계약·개통** — 월/연 구독 선택. 티어 확정.
6. **⑥ 온보딩 체크인** — 도입 후 1주/1개월 CS 점검(cs-support 연계).

> **1인 창업 현실 반영:** 데모·이관은 사람 손이 많이 감. **초기엔 동시 온보딩 2~3곳으로 제한**하고, 반복되는 이관 작업을 템플릿/가이드로 자동화해 나갈 것(액션 항목).

## 5. 신뢰 요소 (후기 없는 초기, 무엇으로 신뢰를 주나)

후기·로고가 없으므로 "제품 자체와 리스크 제거"로 신뢰를 만든다.

1. **무료 체험 + 언제든 해지** — "약정 없음, 가입비 0원"으로 진입 리스크 제거.
2. **데이터 이관 무료 지원** — 이관이 도입의 가장 큰 장벽 → "우리가 옮겨드립니다"가 강력한 신뢰 소구.
3. **데이터 보안·백업 정책 명시** — 회사별 DB 분리(멀티테넌트), 정기 백업. *(주의: 현재 "로그인 없이 API 접근 가능" 이슈가 보류 상태 → 홈페이지에 보안을 강조하려면 **정식 배포 전 인증 적용이 선행**돼야 함. 미해결 상태로 "보안 철저" 카피 쓰지 말 것 — 허위표시 위험.)*
4. **제품 투명성** — 실제 화면 스크린샷·기능 목록·짧은 데모 영상. "실제로 돌아가는 제품"임을 보여주는 게 최고의 신뢰(초기엔 후기보다 이게 낫다).
5. **창업자 직접 응대** — "대표가 직접 도입을 돕습니다" 같은 밀착 지원을 초기엔 강점으로.
6. **얼리버드/베타 고객 혜택** — "초기 고객 특별가 + 기능 우선 반영" → 신뢰라기보다 **초기 유입 인센티브**로 정직하게 포지셔닝.
7. **렌터카 특화 근거 제시** — 차량관리번호 체계, 감가상각·매각·여신 자동전표 등 **도메인 깊이**를 보여주는 것 자체가 "이 사람들 렌터카를 안다"는 신뢰.

> **홈페이지에 쓰면 안 되는 것(허위·과장 방지):** 없는 고객 후기·로고, 검증 안 된 "완전 자동", 미적용된 보안 인증, 미확정 가격을 확정처럼. 전부 전사 원칙(근거·미확인 표시) 위반.
